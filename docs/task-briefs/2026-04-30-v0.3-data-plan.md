# v0.3 data plan -- fix the corpus before training more models

## Why this exists

`v0.2-url-ablation` shipped a URL classifier that flags `https://github.com/anthropics/claude-code` as phishing. Not a model bug -- a corpus shape problem. PhiUSIIL's legit class is 100% bare-domain `https://www.*` URLs with zero path coverage. No URL feature engineering can recover from that. Documented in `LIMITATIONS.md` as the path-presence shortcut.

If we train HTML and screenshot models on PhiUSIIL-shaped scrapes, the same skew shows up in the multi-modal stack: legit pages will systematically be homepages, phish pages will systematically be deep credential forms. The downstream models will learn "homepage shape" instead of "is this trying to steal credentials." Fix data first, train models second.

## Goals

1. **Live phishing pool**, not the stale PhiUSIIL one. Most PhiUSIIL phish hosts are dead Firebase / Cloudways / weebly URLs that 404 when scraped.
2. **Path-bearing legit pool** so the legit class includes URLs that look like real browsing (CDN paths, GitHub permalinks, news article permalinks).
3. **Stratified across URL shape**, not just label. Both classes should have a mix of bare-domain and path-bearing URLs.
4. **Held-out probe sets** that are independent from training. Today's Tranco probe stays. Add an OpenPhish probe disjoint from PhishTank training.

## Non-goals

- Re-doing PhiUSIIL ingestion. We keep PhiUSIIL as one source among many.
- Class rebalancing for absolute size. Quality and shape diversity matter more than count.
- Live Twitter/X URL streams (privacy + ToS).

## Pipeline shape

```
data/raw/
  phishtank/                # cron-pulled JSON, by date

  phiusiil/                 # already there

data/processed/
  url_train.parquet         # rebuilt: union of all sources, deduped, stratified split
  url_val.parquet
  url_test.parquet
  url_probe_tranco.parquet         # held-out, today's logic preserved
  url_probe_openphish.parquet      # NEW: held-out live phish probe
  url_corpus_manifest.json         # provenance per row: which source, when pulled
```

Provenance per row matters. A future review can ask "where did this URL come from" and get an answer.

## Sampling strategy (the part that matters)

Naive concatenation re-creates the PhiUSIIL skew. The legit class has to be **stratified by URL shape**, not just by source.

Define URL shape buckets:

- `bare_domain` -- no path beyond `/`
- `shallow_path` -- 1-2 path segments, no query
- `deep_path` -- 3+ path segments, possibly with query
- `query_heavy` -- path with `?` and 2+ params

Sample legit URLs to roughly match the phish shape distribution **observed in the live phish pool** (PhishTank + OpenPhish), not in PhiUSIIL. PhiUSIIL phish is biased toward Firebase/weebly subdomains; live phish is more URL-shape-diverse.

Target legit shape distribution (rough, will tune):

| Shape | Target % of legit |
|---|---|
| bare_domain | 35 |
| shallow_path | 30 |
| deep_path | 25 |
| query_heavy | 10 |

This is **the actual fix for the github.com false positive.** Once legit URLs include `deep_path` cases at 25%, the model can no longer use `num_slashes` as a phish proxy.

## Code to add

`src/phishguard/data/sources/`
- `phishtank.py` -- pull verified feed via API, write `data/raw/phishtank/YYYY-MM-DD.json`
- `openphish.py` -- pull community feed, dedup against PhishTank
- `commoncrawl.py` -- query CC index for path-bearing URLs from a curated host list (github.com, news sites, docs sites). Use CC because Tranco only gives bare hostnames.
- `tranco.py` -- factor existing logic out of `data/load.py`

`src/phishguard/data/build_corpus.py` -- new orchestrator. Replaces the body of `data/load.py::main`. Reads all sources, dedupes, computes shape buckets, applies stratified sampling, writes splits + manifest.

`tests/test_corpus_shape.py` -- regression test. Asserts:
- Legit class has 20%+ deep_path URLs
- Phish and legit class shape distributions are within 10pp of each other on `bare_domain` (no shape-as-tell)
- No URL appears in both train and any probe set
- Manifest provenance covers 100% of rows

## Risk register

| Risk | Mitigation |
|---|---|
| PhishTank API rate limits or auth churn | Cache aggressively in `data/raw/phishtank/`, add retry+backoff, support env-var key rotation |
| Common Crawl is huge / slow to query | Use the CDX index API, not the WARC files. Filter to specific hosts via prefix queries. |
| Live phish URLs scrape to dead pages by the time training runs | Time-box: scrape within 24hr of pulling the URL. Track dead-rate per source. |
| Source-as-tell shortcut (model learns "is-this-PhishTank") | Stratified sampling in shape buckets exposes this. The shape test catches it. |
| Legal / ToS risk pulling Common Crawl URLs | CC is publicly licensed for research. Document use in `LIMITATIONS.md`. Never re-host content. |

## Sequence (in order, no skipping)

1. **PhishTank ingestion** -- get a key, pull a snapshot, eyeball the URLs, write the source module.
2. **OpenPhish ingestion** -- same pattern. Held out as a probe, not training.
3. **Common Crawl path-bearing legit ingestion** -- this is the heaviest piece. Test against ~10 curated hosts before scaling.
4. **`build_corpus.py`** with the shape-stratified sampler. Write the manifest format.
5. **Shape regression test** before any retraining. The test should run in CI on every PR that touches data.
6. **Retrain v0.3** on the new corpus. Compare v0.2 vs v0.3 on:
   - PhiUSIIL test (sanity, expect v0.3 lower because corpus is harder)
   - Tranco probe FPR (expect v0.3 lower than v0.2's 1.54%)
   - **NEW: Path-bearing legit probe** sampled from CC, expect v0.3 dramatically lower than v0.2's near-100%
   - **NEW: OpenPhish probe** for held-out phish recall
7. **Update MODEL_CARD.md** with v0.3 numbers and revised failure modes.
8. **Tag `v0.3-corpus-rebuild`**, then start HTML/img scraping on this corpus.

## Out-of-scope deliberately deferred

- HTML and screenshot training. They wait until the corpus is right.
- Adversarial examples (FGSM on screenshots). Goes with the screenshot model.
- Multi-language URL coverage. PhiUSIIL skews English/Western; CC sampling can pull from `.cn`, `.ru`, `.in` hosts but this is a v0.4 concern.
